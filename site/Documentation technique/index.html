🚀 Installation et Configuration d'Ansible sur Ubuntu

📌 Prérequis

Avant de commencer, assurez-vous d'avoir :

  • 💻 VirtualBox installé avec 2 VMs Ubuntu (Control Node et Node cible).
  • 🌐 Accès Internet sur les VMs.

🛠 Installation et Configuration

1️⃣ Installer Python 🐍 (nécessaire pour Ansible sur les 2 machines)

✅ Sur Ubuntu/Debian :

sudo apt update
sudo apt install python3

✅ Sur CentOS/RHEL :

sudo yum install python3

2️⃣ Installer Ansible 🛠️ sur la Control Node

sudo apt update
sudo apt install software-properties-common
sudo apt-add-repository --yes --update ppa:ansible/ansible
sudo apt install ansible
ansible --version  # Vérification

3️⃣Privilèges sudo sur la cible 🔑

📌 Remarque : Effectuer cette configuration sur la node (infra1).
L'utilisateur doit avoir des droits sudo pour exécuter des tâches administratives.

➤ Ajouter l'utilisateur au groupe sudo :

✅ Sur Ubuntu/Debian :
sudo usermod -aG sudo your_user
✅ Sur CentOS/RHEL :
sudo usermod -aG wheel your_user

➤ Configurer sudo sans mot de passe :

  1. Éditez le fichier sudoers :

    bash sudo visudo

  2. Ajoutez la ligne suivante :

    bash your_user ALL=(ALL) NOPASSWD: ALL

➤ Aller dans l'inventory si ça ne fonctionne toujours pas :

  1. Éditez le fichier sudoers :

    ```bash 192.168.1.100 ansible_user=your_user ansible_ssh_private_key_file=~/.ssh/id_rsa ansible_become_pass=your_sudo_password

    ```

4️⃣ Installer OpenSSH 🔑 (pour la communication entre les VMs)

✅ Sur Ubuntu/Debian :

sudo apt install openssh-server

✅ Sur CentOS/RHEL :

sudo yum install openssh-server
sudo systemctl start sshd
sudo systemctl enable sshd

5️⃣ Générer une Clé SSH 🔑 sur la Control Node

ssh-keygen -t rsa -b 2048  # Appuyez sur Entrée pour les valeurs par défaut

6️⃣Copier la Clé SSH depuis la Control Node vers le Node Cible 📤

ssh-copy-id user@ip_cible  # Remplacez 'user' et 'ip_cible'
ssh user@ip_cible  # Vérifiez l'accès SSH sans mot de passe

7️⃣ : Relier la Control Node au VS Code de la machine hôte 💻

Pour pouvoir travailler directement depuis votre éditeur de texte (VS Code) sur votre machine hôte, vous devez configurer l'accès à la Control Node via SSH.

  1. Installer l'extension Remote - SSH dans VS Code :

    • Ouvrez VS Code et allez dans le Marketplace.
    • Recherchez l'extension Remote - SSH et installez-la.

    • Configurer la connexion SSH :

    • Dans VS Code, cliquez sur l'icône Remote Explorer (en bas à gauche) puis sur Add New SSH Host.

    • Entrez la commande SSH pour vous connecter à la Control Node :

      bash ssh user@ip_control_node # Remplacez 'user' et 'ip_control_node'

    • Vous pouvez aussi configurer cela directement dans le fichier ~/.ssh/config sur votre machine hôte :

      bash Host control-node HostName ip_control_node User your_user IdentityFile ~/.ssh/id_rsa # Si nécessaire

  2. Se connecter à la Control Node depuis VS Code :

    • Une fois la configuration terminée, vous pouvez cliquer sur Remote-SSH: Connect to Host et choisir control-node.
    • Vous serez maintenant connecté à la Control Node via VS Code et pourrez l'utiliser comme si vous y étiez.

8️⃣ : Créer un Dossier pour le Playbook dans la Control Node depuis VS Code 🗂️

  1. Créer un nouveau dossier pour votre Playbook :

    • Une fois connecté à la Control Node depuis VS Code, ouvrez un terminal intégré dans VS Code (Terminal > New Terminal) ou le faire graphiquement.

    • Créez un dossier pour votre projet de playbook, par exemple :

      bash mkdir ~/ansible_playbooks cd ~/ansible_playbooks

  2. Créer votre premier playbook Ansible :

    • Dans le dossier que vous venez de créer, créez un fichier YAML pour votre playbook. Par exemple, créez un fichier first_playbook.yml :

      bash touch first_playbook.yml

    • Ouvrez ce fichier dans VS Code et commencez à écrire votre playbook. Exemple de contenu basique :

      ```yaml

      • name: Install Apache hosts: all become: yes tasks:
        • name: Install apache2 package apt: name: apache2 state: present ```

  3. Exécuter le Playbook depuis VS Code :

    • Vous pouvez exécuter ce playbook directement depuis le terminal intégré de VS Code :

      bash ansible-playbook first_playbook.yml

🎯 Vous êtes prêt !
Vous avez maintenant relié votre Control Node à VS Code de la machine hôte et créé un dossier pour y stocker vos playbooks Ansible. Vous pouvez ainsi facilement développer, éditer et exécuter vos playbooks depuis VS Code de la machine hôte. 🚀
